Project eevee-helper - v0.0.1-rc.6

工具函数

函数仓库是 Eevee 生态下，函数复用的主要方式。函数仓库中，主要以业务逻辑为前提来实现抽象复用，上层不需要关注 API 内部的逻辑，只对外提供接口

命令

 
# 构建文档
pnpm build:docs

# 发布文档
pnpm publish:docs

# 发布工具库
pnpm publish:lib

使用&扩展

一、base 基础功能

1. http 请求

初始化配置

setHttpConfig({
    timeout: 5000, // 超时（默认5000）
    retries: 2, // 重试次数（默认0，该配置针对所有接口生效）
    retryDelay: () => 0, // 重试延迟
    interceptors: {
        request: (config) => {
            // 处理config
            ...
            return config;
        },
        requestError: (err) => err,
        response: (res) => res,
        responseError: (err) => err,
    },
});

常规使用:

import { http } from '@peiyou/eevee-helper';

http.get('api.peiyou.com', { aaa: '123' });

自定义 headers

http.get(
    'api.peiyou.com',
    { aaa: '123' },
    {
        headers: {
            token: 'xxxxxxxx',
        },
    }
);

针对单个接口的自动重试

// 2 为重试两次，加上首次请求总计为3次
http.get(
    'api.peiyou.com',
    { aaa: '123' },
    {
        'axios-retry': {
            retries: 2,
        },
    }
);

2. broadcast 广播事件

适用于多窗口下的消息通知场景

import { broadcast } from '@peiyou/eevee-helper';

// 注意：broadcast使用前提需要每个窗口都先调用initBroadcast
initBroadcast('mix_teacher');

const eventType = {
    GLOBAL_REFRESH: 'GLOBAL_REFRESH',
};

// 派发事件
broadcast.dispatch(eventType.GLOBAL_REFRESH, '刷新消息');

broadcast.on(eventType.GLOBAL_REFRESH, (res) => {
    // res - '刷新消息'
});

3. storage 缓存对象

默认使用 localStorage 作为缓存载体

import { storage } from '@peiyou/eevee-helper';

// 设置缓存并添加过期时间（以小时为单位）
storage.set('token', 'xxxxxxxx', 2);

// 获取缓存
storage.get('token');

// 清除缓存
storage.remove('token');

二、service 业务功能

业务侧主要分为 native 部分及 pomelo 部分

1. native-api

目前 native 按功能划分为以下几大类：

  • 公共部分
  • 推拉流
  • 初始化
  • 窗口操作

如果需要新增功能类别，需要在 native-api 下新创建功能类别文件开发，比如说要开发设备相关 api，需要在 native-api 下新建 device.ts 文件。以获取摄像头列表为例

// src/service/native-api/device.ts

import zby from 'zby-live-sdk';
import { promiseErrorHandle } from './util';

export const getCameraDeviceList = () => {
    return promiseErrorHandle(async () => {
        return await zby.getCameraDeviceList();
    });
};

注意：promiseErrorHandle 方法是内部用于统一错误处理的函数

然后需要在 index 下导出该文件

// src/service/native-api/index.ts

export * from './device';

2. pomelo

pomelo 的状态及事件主要通过 hooks 提供

import { pomelo, usePomelo } from '@peiyou/eevee-helper';

// 首先通过初始化 pomelo 主讲频道内部状态和事件
pomelo.initPomeloChannel(
    {
        host: 'xxxxxxxxxxxx',
        port: 443,
        userID: 'xxxxxxx',
        channelID: 'xxxxxxxx',
        userName: 'xxxxxxx',
        roleID: 4, // 1: 学生；2：辅导老师    4 授课
        classID: xxxxxxx, // liveId
        protocolVersion: '1.0',
        interactMode: 1,
        mainTeacherClientVer: '0.0.0.0',
        uniqId: 'xxxxxxxxxxxxxxxxxxxxx',
        rtype: 1,
    },
    'main'
);

// 使用主讲频道
const { connectStatus, serveMsg } = usePomelo('main');
// 监听下行消息
watch(serveMsg, (val) => {
    // ...
});

新增频道

需要在 src/enums/socketEnum.ts 下 PomeloChannelEnum 和 PomeloChannelKeyEnum 扩展频道

export enum PomeloChannelEnum {
    LECTURER_TEACHER = 'LECTURER_TEACHER',
    TUTOR_TEACHER = 'TUTOR_TEACHER',
    GROUP_TEACHER = 'GROUP_TEACHER',
    TUTOR_SOLO = 'TUTOR_SOLO',

export enum PomeloChannelKeyEnum {
    main = 'LECTURER_TEACHER',
    tutor = 'TUTOR_TEACHER',
    group = 'GROUP_TEACHER',
    solo = 'TUTOR_SOLO',
}

三、hooks

以 commonInfo 使用为例

import { useCommonInfo } from '@peiyou/eevee-helper';

const { clientVersion, winMode, guid, systemName, rtcPullAddress } = useCommonInfo();

如何开发

目前 hooks 大多使用场景都集中在业务数据上，而工具库的业务数据统一由 database 管理，hooks 直接拿来消费。以 useCommonInfo 为例，比如我们只使用客户端版本号

import { computed } from 'vue';
import { database } from '../service/database';

export default function useCommonInfo() {
    const clientVersion = computed(() => database.systemState.version);

    return {
        clientVersion,
    };
}

而客户端版本号是由大班 sdk 提供的，逻辑如下：

// src/service/native-api/common
import pyzx from 'pyzx-live-sdk';

export function getSystemInfo(): Promise<object> {
    return promiseErrorHandle(async () => {
        const result = await pyzx.getSystemInformation();
        sendMsgSyncStore('systemState', result.msg.version, 'version');
        return result.msg;
    });
}
// {
//     msg: {
//         "version":"xxxxxxxx",
//         "systemName":"xxxxxxxx",
//         "guid": "xxxxxxx",
//         "time":"xxxxxxx",
//     }
// }

注意：sendMsgSyncStore 方法是工具库内部用来做数据同步的方法

database 结构如下：

export const database: Database = reactive({
    ...,
    systemState: {
        version: '',
        guid: '',
        systemName: '',
    },
});

# 工具库路径
pnpm link ../eevee/packages/helper
